What's Responsive Web Design?

Responsive Web Design (RWD) is a Design Strategy that aims to provide a deeply contextualized experience for the user regardless of the consumption device that is currently being used to access our site.

Mobile Design is centered on devices.

≠

Responsive Design is centered on context.


This talk is about
The Web.

Responsive Design is also applicable to Native Mobile Apps.

(That's another talk.)

Some Basics:

  • You don't know which device the user is going to use, you aim to cover them all.
  • Make your html Semantic, allow for fallbacks.
  • If is not going to be displayed, don't even download it.
  • Horizontal scrolling is the enemy.
  • It's a lot of work, decide wisely...

RWD is for:

  • Simple Web Sites.

    Complicated applications with customized workflows are better designed as a separate mobile app.

  • Modern Web Sites.

    Responsive on IE6, Symbian? Are you kidding me?

  • Ubiquitous Web Sites.

    If no one is going to access your site on a phone or tablet or tv... why bother?

The pillars of RWD:

  • Feature Detection.

  • Media Queries.

  • Fluid Grids.

  • Responsive Images.

Fluid Grids

Remember when we used tables for layout?

Enter the grid:

A grid-based layout lets us distribute content in columns and rows, just like with a table.

But:

  • Grids grow or shrink following screen size.
  • Grids fallback nicely on small screens.
  • Grids are semantically neutral (Tables should be used for tabular data).
  • Grids are easily extensible and customizable.
